# ThreeJSON 特性

ThreeJSON 是一个 JSON 驱动的 Three.js 场景运行时。它把场景描述为结构化数据，让同一份场景可以被运行时加载、被工具编辑、被示例复用，也可以进入 AI 生成和增量修改流程。

## JSON 驱动

场景不再必须写成大量命令式 JavaScript。JSON 可以描述渲染器、相机、控制器、灯光、对象、材质、动画、事件和业务域对象。

## AI 友好

场景是结构化数据，LLM 更容易生成、检查、解释和修补。ThreeJSON 支持全量场景 JSON、增量 JSON patch 和面向运行时变更的命令式更新。

## 对象管理

有意义的对象可以拥有稳定的 `threeJsonId`、`name`、`label` 和类型化对象记录。这让选中、高亮、编辑、导出和运行时变更都更可靠。

## Domains

Domain handler 可以把更高层的业务概念表达为 JSON，同时部署为 Three.js 对象。它适合业务组件、拓扑对象、面板、图表和可复用场景模式。

## 扩展机制

扩展提供可选能力，不把所有功能塞进核心运行时。例如第一人称导航、物理、进阶渲染、CSS3D 面板等都可以作为可选能力接入。

## 运行时变更

ThreeJSON 支持运行时对象变更和声明式命令，工具可以局部更新场景而不必整体重建。这也是编辑器操作、AI 增量调整和交互式示例的基础。

## 事件与脚本

事件机制把 JSON 对象连接到声明式动作和可选脚本，支持 click、hover、生命周期和自定义行为，同时保持场景定义的可移植性。
